# Security Policy

## Supported Versions

The following versions of MusicDP are currently supported with security updates and bug fixes.

| Version | Supported |
| ------- | --------- |
| 1.2.x   | ✅ Yes |
| 1.1.x   | ✅ Yes |
| 1.0.x   | ⚠️ Limited Support |
| < 1.0   | ❌ No |

We recommend all users upgrade to the latest version to receive security updates and improvements.

---

## Reporting a Vulnerability

If you discover a security vulnerability in MusicDP, please report it responsibly.

### How to Report
Please send details of the vulnerability to:

**Email:** ddprasad@example.com

Include the following information if possible:

- Description of the vulnerability
- Steps to reproduce the issue
- Affected version of the application
- Device / Android version (if applicable)
- Screenshots or logs if available

### Response Time
- We aim to acknowledge vulnerability reports within **48 hours**.
- A fix or mitigation plan will usually be provided within **7–14 days**, depending on the severity of the issue.

### Responsible Disclosure
Please **do not publicly disclose** the vulnerability until it has been investigated and resolved. Responsible disclosure helps protect users while a fix is being prepared.

### Security Updates
Security patches will be released through the project's repository and application updates distributed via the app store.
